# Локальный GitLab с помощью Vagrant и Hyper-V

Данный проект позволяет быстро развернуть полнофункциональный сервер GitLab Community Edition (CE) в локальной среде Windows с использованием виртуализации Hyper-V.

## 📋 Особенности сборки

- **Сетевой IP:** ВМ разворачивается через `public_network`. IP-адрес выдается вашим роутером по DHCP.
- **GitLab CE:** Установлена бесплатная версия GitLab с поддержкой Omnibus.
- **Функции:** 
  - **Wiki:** Доступна для всех проектов.
  - **Package Registry:** Включена поддержка пакетных реестров (Maven, NuGet, NPM, Generic, PyPI).
- **Автоматизация:** В комплекте идет скрипт для автоматического обновления `hosts` и создания ярлыка на рабочем столе.

## 💻 Требования

- **Windows 10/11** с включенным компонентом Hyper-V.
- **Vagrant 2.4.7+** (рекомендуется последняя версия).
- **8 ГБ свободной ОЗУ** (GitLab требует значительных ресурсов для стабильной работы).
- **PowerShell** (запускать от имени администратора).

## 🚀 Шаги установки

1.  **Клонируйте/скачайте** этот репозиторий.
2.  **Откройте PowerShell с правами администратора** в папке проекта.
3.  **Запустите развёртывание:**
    ```powershell
    vagrant up
    ```
    - **Важно:** Процесс распаковки (Unpacking) и настройки может занять **10-20 минут**. Если кажется, что процесс завис на 1.4 ГБ — просто подождите.

## 🌐 Доступ к GitLab

### Способ А: Автоматический (Рекомендуется)
После завершения `vagrant up`, выполните скрипт `update-gitlab-host.ps1`:
1. Нажмите правой кнопкой на `update-gitlab-host.ps1` -> **"Run with PowerShell"**.
2. Скрипт сам узнает IP, пропишет его в `C:\Windows\System32\drivers\etc\hosts` для домена `http://gitlab.local` и создаст ярлык на рабочем столе.

### Способ Б: Ручной
1. **Узнайте IP:** `vagrant ssh -c "hostname -I"`
2. **Откройте браузер:** Перейдите по полученному адресу (например, `http://192.168.1.x`).

### Пароль администратора (root)
При первой установке GitLab генерирует пароль. Узнать его можно командой:
```powershell
vagrant ssh -c "sudo cat /etc/gitlab/initial_root_password"
```
*Внимание: Смените пароль при первом входе! Файл удалится через 24 часа.*

## ⚙️ Управление ВМ

Команды выполняются из папки проекта:
- `vagrant status` — проверить состояние.
- `vagrant halt` — корректно выключить.
- `vagrant up` — включить.
- `vagrant reload` — перезагрузить (применяет изменения в `Vagrantfile`).
- `vagrant ssh` — войти в консоль Ubuntu.

## 📦 Перенос и бэкап

Для переноса на другой ПК просто остановите машину (`vagrant halt`) и скопируйте всю папку проекта на флешку. На новом ПК выполните `vagrant up`. Подробнее см. в [docs/storage.md](docs/storage.md).
