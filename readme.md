# Локальный GitLab с помощью Vagrant и Hyper-V

## Требования
- Windows 10/11 с Hyper-V
- Vagrant 2.4.7+ (под вашу систему)
- Powershell (для выполнения команд)
- Ubuntu 22.04 LTS (Jammy Jellyfish) (используется box `generic/ubuntu2204`)

## Особенности сборки
- **Сетевой IP:** ВМ разворачивается через `public_network`. IP-адрес выдается вашим роутером по DHCP.
- **GitLab CE:** Установлена бесплатная версия GitLab.
- **Функции:** 
  - **Wiki:** Доступна для проектов.
  - **Package Registry:** Включена поддержка пакетных реестров (Maven, NuGet, NPM, Generic).

## Шаги установки

1. **Клонируйте/скачайте этот репозиторий**
2. **Откройте Powershell с правами администратора**
3. **Перейдите в папку проекта**
   ```powershell
   cd путь\к\папке\Gitlab
   ```
4. **Запустите развёртывание**
   ```powershell
   vagrant up
   ```
   - **Важно:** Процесс распаковки (Unpacking) и настройки GitLab может занять от **10 до 20 минут**. Если кажется, что процесс «завис» на распаковке пакета весом 1.4 ГБ — просто подождите.

## Доступ к GitLab

1. **Узнайте IP-адрес:** 
   Если в конце установки IP не вывелся автоматически, выполните:
   ```powershell
   vagrant ssh -c "hostname -I"
   ```
2. **Откройте браузер:** Перейдите по полученному адресу (например, `http://192.168.1.x`).
3. **Получите временный пароль root:**
   При первой установке GitLab генерирует пароль для администратора. Узнать его можно командой:
   ```powershell
   vagrant ssh -c "sudo cat /etc/gitlab/initial_root_password"
   ```
   - **Внимание:** Обязательно смените пароль при первом входе! Этот файл с паролем удалится через 24 часа.

## Управление виртуальной машиной

Все команды выполняются из папки проекта:
- **Узнать статус:** `vagrant status`
- **Остановить VM (корректно):** `vagrant halt`
- **Запустить VM:** `vagrant up`
- **Перезагрузить VM:** `vagrant reload`
- **Подключиться по SSH:** `vagrant ssh`

## Автоматизация настройки (рекомендуется)

В корне проекта находится PowerShell скрипт `update-gitlab-host.ps1`, который автоматизирует настройку доступа.

### Что делает скрипт:
1.  **Получает IP:** Автоматически запрашивает текущий IP-адрес у запущенной виртуальной машины.
2.  **Обновляет hosts:** Удаляет старую запись для `gitlab.local` и добавляет актуальную в `C:\Windows\System32\drivers\etc\hosts`.
3.  **Создает ярлык:** Размещает ярлык `gitlab` на вашем рабочем столе для быстрого запуска.
4.  **Открывает браузер:** Сразу переходит на `http://gitlab.local`.

### Как использовать:
1.  Убедитесь, что виртуальная машина запущена (`vagrant up`).
2.  Нажмите правой кнопкой мыши на файл `update-gitlab-host.ps1` и выберите **"Run with PowerShell"** (Выполнить с помощью PowerShell).
3.  **Важно:** Скрипт запросит права администратора (UAC), так как редактирование файла `hosts` требует повышенных привилегий.
4.  После первого запуска вы можете использовать ярлык **gitlab** на рабочем столе.

---

## Примечания
- **Доменное имя:** Чтобы заходить по адресу `http://gitlab.local` вручную, добавьте строку `ваш_ip gitlab.local` в hosts-файл Windows (`C:\Windows\System32\drivers\etc\hosts`).
- **Изменение IP:** В данной сборке используется DHCP. Если вам нужен статический IP, настройте резервирование адреса на вашем роутере для MAC-адреса ВМ.
